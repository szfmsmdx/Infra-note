## new/delete 与 malloc/free
- **`new`**: 在分配内存后，会自动调用对象的**构造函数**来初始化对象。
- **`delete`**: 在释放内存前，会自动调用对象的**析构函数**来清理对象资源。
- **`malloc/free`**: 仅仅负责分配和释放指定大小的原始内存块，**完全不会**调用构造函数或析构函数。

new = operator new 分配内存 + 调用构造函数
## 栈、堆、静态区、常量区。哪些是编译期决定的？
“静态区”和“常量区”在编译期决定（布局已定），而“栈”和“堆”是在运行期决定的。

|内存区域|决定时机|管理方式|典型内容|备注|
|:--|:--|:--|:--|:--|
|静态区|编译期|系统自动|全局变量、`static` 变量|程序启动即存在，结束才释放。|
|常量区|编译期|系统自动|字符串常量 (`"hello"`)、`const` 全局变量|通常只读，放在 `.rodata` 段。|
|栈区|运行期|编译器自动|局部变量、函数参数|函数调用时分配，返回时释放。|
|堆区|运行期|程序员手动|`new`/`malloc` 的对象|动态分配，容易产生碎片和泄漏。|

虽然栈的**分配动作**发生在运行期（函数调用时），但栈帧的大小通常是在**编译期**就已经计算好的（除非是变长数组 VLA）。
- **比如**：`func()` 里有 `int a, b, c`，编译器在编译时就知道 `func` 的栈帧需要 12 字节。

## RAII？
就是 “把资源的生命周期绑定在对象的生命周期上”。
- 核心逻辑：
    - 构造时（出生）：在构造函数里申请资源（内存、文件句柄、锁）。
    - 析构时（死亡）：在析构函数里自动释放资源。

## 智能指针？
核心思想是 RAII，在构造函数中申请资源，在析构函数中释放资源，因为局部对象在离开作用域时会自动调用析构函数（哪怕发生异常），所以资源永远不会泄漏

| 智能指针         | 所有权模式 | 性能/特点                | 典型场景                      |
| :----------- | :---- | :------------------- | :------------------------ |
| `unique_ptr` | 独占    | 零开销，不可复制，只能移动。       | 绝大多数场景的首选。                |
| `shared_ptr` | 共享    | 内部维护引用计数，线程安全但有性能损耗。 | 需要多个指针指向同一对象时。            |
| `weak_ptr`   | 弱引用   | 不占有资源，只观察。           | 解决 `shared_ptr` 的死循环引用问题。 |

### unique_prt
禁止拷贝，防止资源被两份释放
```cpp
#include <memory> 
#include <iostream> 
void testUnique() { 
	// C++14 推荐用 make_unique 
	std::unique_ptr<int> p1 = std::make_unique<int>(10); 
	std::cout << *p1 << std::endl; // 输出 10 
	
	// std::unique_ptr<int> p2 = p1; // 报错！不能拷贝 
	std::unique_ptr<int> p2 = std::move(p1); // 正确！所有权转移 
	// 此时 p1 变为空指针，p2 接管内存 
} // p2 离开作用域，内存自动释放
```

### shared_ptr
通过引用计数管理，计数归零才释放。
```cpp
void testShared() { 
	std::shared_ptr<int> p1 = std::make_shared<int>(20); 
	{ 
		std::shared_ptr<int> p2 = p1; // 拷贝，引用计数 +1 (变为2) 
		std::cout << p1.use_count() << std::endl; // 输出 2 
	} // p2 离开作用域，引用计数 -1 (变为1)，内存不释放 
} // p1 离开作用域，引用计数 -1 (变为0)，内存自动释放
```

### weak_ptr
如果两个对象互相持有 `shared_ptr`，引用计数永远降不到 0，导致内存泄漏。例如：
```cpp
#include <memory>
#include <iostream>

class B; // 前置声明

class A {
public:
    std::shared_ptr<B> b_ptr; // A 持有 B
    ~A() { std::cout << "A 析构" << std::endl; }
};

class B {
public:
    std::shared_ptr<A> a_ptr; // B 持有 A (问题出在这里！)
    ~B() { std::cout << "B 析构" << std::endl; }
};

void testCycle() {
    auto a = std::make_shared<A>();
    auto b = std::make_shared<B>();
    a->b_ptr = b; // A 指向 B，B 的计数变 2
    b->a_ptr = a; // B 指向 A，A 的计数变 2
} // 函数结束，a 和 b 离开作用域，计数各减 1，但还剩 1（互相持有）
  // 内存泄漏！A 和 B 的析构函数都不会被调用。

int main() {
    testCycle();
    std::cout << "程序结束" << std::endl;
    return 0;
}
```

所以要用 weak_ptr 解决这个相互引用的问题， weak_ptr 不增加引用
```cpp
// 假设 A 和 B 互相持有 
// A 中存 shared_ptr<B> 
// B 中存 weak_ptr<A> (不增加引用计数) 
void testWeak() { 
	std::shared_ptr<int> p1 = std::make_shared<int>(30); 
	std::weak_ptr<int> wp = p1; // weak_ptr 指向 shared_ptr 
	
	if (!wp.expired()) { // 使用前必须 lock() 提升为 shared_ptr 
		std::shared_ptr<int> p2 = wp.lock(); 
		std::cout << *p2 << std::endl; 
	}
}
```


## 左值引用和右值引用？
首先说一下左值和右值
- 左值：有名字、有固定内存地址、生命周期持久的对象
- 右值：临时的、无名的、即将销毁的数据

一般来说，cpp 里面是这样的，对于 int c = a + b，这个表达式，那么 a + b 是临时创建的右值，需要先构造再深拷贝给 c，这个时候是不划算的，所以就有左值引用和右值引用的概念：
- 左值引用：比如 int& a，能够直接拿到左值的地址
- 右值引用：比如 int&&，这就是拿到右值引用，而不需要经过拷贝的过程

移动语义是通过 std:move 来实现的，它只是一个类型转换，把一个左值（有名字的变量）强制转换成一个右值引用。
```cpp
#include <iostream> 
#include <vector> 
#include <utility> // std::move 的头文件 
int main() { 
	std::vector<int> v1 = {1, 2, 3}; 
	std::cout << "v1 地址: " << &v1 << std::endl; 
	
	// 方式一：拷贝 (深拷贝，开销大) 
	// std::vector<int> v2 = v1; 
	
	// 方式二：移动 (窃取资源，开销极小) 
	std::vector<int> v2 = std::move(v1); 
	std::cout << "v2 地址: " << &v2 << std::endl; 
	// v1 的资源被“偷”走了，现在 v1 处于有效但未定义的状态（通常是空的） 
	
	std::cout << "v1 大小: " << v1.size() << std::endl; // 输出 0 
	std::cout << "v2 大小: " << v2.size() << std::endl; // 输出 3 
	return 0; 
}
```

一般来说用了 move 之后，原始对象就处于一个未定义的状态了
```cpp
std::string s1 = "Hello"; 
std::string s2 = std::move(s1); // s1 的资源被移动到 s2 

// ❌ 危险！s1 的内容现在是未定义的（通常是空字符串） 
std::cout << s1 << std::endl; // 可能输出空行，行为未定义
```

然后对于 T&& 的话，他其实有两个作用：
- 右值引用（没有类型推导的情况下）：
```cpp
void func(int&& ref) { 
	// ref 是右值引用 
	// 只能接收临时对象 
} 

int main() { 
	int a = 10; 
	func(10); // ✅ OK：10 是右值 
	func(a); // ❌ 编译错误！a 是左值，不能传给 int&& 
	func(std::move(a)); // ✅ OK：std::move 把 a 变成了右值 
}
```

- 万能引用（有类型推导的情况下，比如模板或者 auto）：
```cpp
template<typename T> 
void wrapper(T&& param) { 
	// 这里的 T&& 是万能引用 
} 

int main() { 
	int a = 10; 
	wrapper(a); // 传左值 -> T&& 变身成为 左值引用 (int&) 
	wrapper(10); // 传右值 -> T&& 变身成为 右值引用 (int&&) 
}
```
- 传左值 `a` 时：编译器推导 `T` 为 `int&`。于是 `T&&` 变成了 `int& &&`。根据 C++ 规则，`& &&` 折叠为 `&`。所以 `param` 是 `int&`。
- 传右值 `10` 时：编译器推导 `T` 为 `int`。于是 `T&&` 变成了 `int&&`。所以 `param` 是 `int&&`。
