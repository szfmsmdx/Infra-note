## 封装、继承和多态
### 封装
把数据（属性）和操作数据的方法（行为）捆绑在一起，形成“类”，以隐藏实现细节，保护数据安全，例如：
```cpp
class User { 
	private: // 隐藏数据 
		string password; 
	public: // 暴露接口 
		void login(string p) { 
			if(p == password) cout << "Login OK"; 
		} 
	};
```

### 继承
子类（派生类）继承父类（基类）的属性和方法，语法：`class Child : public Parent` 
基类的 `private` 成员被继承了但**不可见**（无法直接访问）

### 多态
父类指针调用子类的方法
实现方式：
1. **静态多态（编译时）**：通过**函数重载**、**模板**实现。
2. **动态多态（运行时）**：必须满足两个条件：
	- 基类函数必须加 `virtual` 关键字（变成虚函数）。
	- 通过基类的**指针**或**引用**调用该函数。

静态多态通过模板或者重载的方式去实现：
```cpp
#include <iostream> 
using namespace std; 

// 模板函数：编译器会根据传入类型 T 自动生成代码 
template <typename T> 
void show(T val) { 
	cout << "Value: " << val << endl; 
} 

int main() { 
	show(10); // 编译器实例化 show<int> 
	show("Hi"); // 编译器实例化 show<const char*> 
	return 0; 
}
```

动态多态的话通过 virtual 关键字去实现
```cpp
class Base { 
public: 
	// 必须加 virtual 才能实现多态 
	virtual void show() { cout << "Base"; } 
}; 

class Derived : public Base { 
public: void show() override { cout << "Derived"; } // 重写 
}; 

// 调用 
Base* ptr = new Derived(); 
ptr->show(); // 输出 "Derived" (如果没有 virtual，则输出 "Base")
```

注意：
1. 子类有自己的虚函数表，存着虚函数的地址
2. 每个对象有虚函数指针，指向子类的虚函数表
3. 调用时对象通过 vptr 找到 vtable 拿到正确的虚函数地址

## 虚函数与纯虚函数

纯虚函数的话，就是在虚函数声明上加上 =0
```cpp
virtual void func() = 0;
```

那么这个时候，含有纯虚函数的类就是抽象类，不能创建对象
派生类必须强制实现这个纯虚函数，否则派生类也是抽象类
纯虚析构函数很特殊，它**必须在类外提供定义**。因为对象销毁时，析构函数总会被调用，如果没有定义，链接时会报错

## CRTP（Curiously Recurring Template Pattern）？它如何替代运行时多态？

参考这个例子：
```cpp
#include <iostream>
using namespace std;

// 基类是模板，把自己作为模板参数传进去
template <class Derived>
class Base {
public:
    void interface() {
        // 静态转型，编译时确定调用 Derived::impl()
        static_cast<Derived*>(this)->impl();
    }
};

class Child : public Base<Child> {
public:
    void impl() { cout << "Static Polymorphism via CRTP" << endl; }
};

  
int main() {
    Child c;
    c.interface(); // 输出 "Static Polymorphism via CRTP"
    return 0;
}
```
它能让基类在编译时知道派生类的类型，因为 static_cast 是静态转型，编译的时候就需要确定了，所以也会有一个编译时检查的一个作用
