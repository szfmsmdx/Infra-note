# 进程、线程、协程有什么区别？
- 进程是OS分配资源的最小单位，他的通信需要用shm、管道等机制实现，用多进程是可以绕过GIL机制的
- 线程的话，是OS调度的最小单位，是属于进程的，共享进程堆、变量等一些信息，但是有独立的寄存器等资源，但受限于GIL，一些计算密集没法通过线程去多核执行
- 协程是用户态线程，由框架（如async）调度，共享线程的资源，用来处理一些并发请求

一般使用的话，采用多进程管多卡，多线程IO和协程去异步处理一些并发请求

# 三者是怎么切换的？
- 进程切换需要涉及页表，缓存会失效，开销成本很高
- 线程切换需要恢复寄存器上下文，不需要恢复地址空间
- 协程切换涉及一些栈指针和少量寄存器保存，具体流程为：
  - 保护现场，维护寄存器状态到线程控制块（TCB）中
  - 更新数据结构，比如更新协程状态
  - 恢复现场，读取要切换到的协程状态
  - 返回用户态开始执行命令
  - 开销包括：
    - 寄存器保存、恢复 —— cpu 指令开销
    - 缓存会失效
    - 流水线冲刷 —— 分支预测失败导致 CPU 流水线清空
    - 内存态切换
